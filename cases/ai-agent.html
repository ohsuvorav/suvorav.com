Context

The team was good. Its knowledge wasn't portable.

The platform team had real delivery accountability — but everything that made it work lived in people's heads. Decisions, edge cases, operational patterns: in Slack threads, in memory, nowhere shareable. When an engineer was out, things stalled. When a new one joined, ramp was measured in months.

My mandate: make the team faster without adding headcount. The answer wasn't more engineers — it was making the team's knowledge a first-class citizen.

Problem

Four separate problems. One root cause.

Knowledge siloed — context sharing was manual and lossy; AI agents couldn't reference what only existed in someone's memory. No automation — Linear has no native triggers (unlike Jira), so every routing and triage decision was human. Invisible tradeoffs — deprioritizing a ticket hid what else would slip as a consequence. Expensive debugging — every incoming bug required 2–3 hrs of context reconstruction before the actual work could start.

These weren't four separate problems. They shared a root cause: the team's operating context was stuck in people's heads.
What I built

Not four solutions. One compound system.

Team brain in Git — Every workflow, decision, and operating practice committed as .md files. A custom Claude skill auto-updated them after each session: if something changed, the docs reflected it by the next morning. New engineers onboard by reading the files. AI agents reference them without asking anyone.

Linear Triggers — Jira has native triggers. Linear doesn't. Built webhook-based automations: auto-route on filing, auto-ping on stale tickets, surface blocked dependencies automatically. Triage stopped requiring meetings.

Capacity planner — When something got deprioritized, the downstream impact on everything else in flight became visible before the decision was made. Tradeoffs were no longer blind.

[Capacity planner screenshot]

Claude wrapper — Incoming Linear bugs auto-generate reproduction steps, environment context, and a root cause analysis template — pre-filled from the ticket data and similar past issues. An engineer opens a bug and 80% of the investigation is already scaffolded.

Impact

2.5× bandwidth. Zero new hires.

Engineering capacity doubled without adding a single hire. Three things drove it:

EM delegation — mentored the engineering manager on structured handoff: architect defines outcome, engineer owns execution. Eliminated the single-point-of-failure where everything routed through one person.

Predictable delivery — centralized Linear intake + automatic routing + visible capacity removed the invisible coordination tax that was eating sprint after sprint.

2–3 hrs freed per engineer per day — the AI agent handled investigation scaffolding that used to be done manually. Multiply that across the team and it's a significant chunk of the 2.5× number.

The compound effect: the team brain keeps improving. Each session updates the docs, each future session starts with more context. It's not a one-time gain — it compounds.
